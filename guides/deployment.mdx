---
title: "Build and deploy PubKey UI Starter with Docker"
description: "Build the PubKey UI Starter for production, package the API with Docker, and deploy the React frontend as static files served by the Express server."
---

In production, the Express API server doubles as a static file host for the compiled React app. The web app is built first, its output is copied into the API's asset directory, and then the entire stack is packaged into a single Docker image. This means one container serves both the REST API under `/api` and the React frontend under `/assets`.

## Production build

<Steps>
  <Step title="Build the web app">
    Compile and bundle the React application with webpack:

    ```bash
    pnpm build:web
    ```

    Output is written to `dist/web/`. The production build enables full optimization: minification, output hashing, and license extraction.
  </Step>

  <Step title="Build the API">
    Compile the Express server with esbuild:

    ```bash
    pnpm build:api
    ```

    Output is written to `dist/api/`. The build bundles all dependencies, generates a `package.json` and lockfile for the Docker image, and produces ESM output.
  </Step>

  <Step title="Copy static assets">
    The API build copies the contents of `api/src/assets/` into `dist/api/assets/`. The Express server serves this directory at startup. To include the compiled React app, copy the output of `dist/web/` into `api/src/assets/` before running `pnpm build:api`, or add this step to your CI pipeline.

    ```bash
    cp -r dist/web/* api/src/assets/
    pnpm build:api
    ```
  </Step>
</Steps>

<Warning>
  The Express server calls `process.exit(1)` at startup if the `dist/api/assets/` directory does not exist. Always build the web app before building the API, or use the Docker build target which handles the dependency order automatically.
</Warning>

## Docker deployment

The `api` project includes a Dockerfile generated by Nx. Build the image using the Nx `docker-build` target, which first runs the API build and then invokes Docker.

### Dockerfile

```dockerfile api/Dockerfile
# This file is generated by Nx.
#
# Build the docker image with `npx nx docker-build api`.
# Tip: Modify "docker-build" options in project.json to change docker build args.
#
# Run the container with `docker run -p 3000:3000 -t api`.
FROM docker.io/node:lts-alpine

ENV HOST=0.0.0.0
ENV PORT=3000

WORKDIR /app

RUN addgroup --system api && \
          adduser --system -G api api

COPY dist/api api
RUN chown -R api:api .

# You can remove this install step if you build with `--bundle` option.
# The bundled output will include external dependencies.
RUN npm --prefix api --omit=dev -f install

CMD [ "node", "api" ]
```

### Build the image

```bash
npx nx docker-build api
```

This runs the `docker-build` target from `api/project.json`, which depends on `api:build` — so the TypeScript compilation step runs automatically before the Docker build.

### Run the container

```bash
docker run -p 3000:3000 -t api
```

The server starts on port 3000 and is accessible at `http://localhost:3000`.

## Environment variables

Configure the server at runtime by passing environment variables to `docker run`:

| Variable | Default | Description |
|---|---|---|
| `PORT` | `3000` | Port the Express server listens on |
| `HOST` | `0.0.0.0` | Network interface the server binds to |
| `API_URL` | `http://{HOST}:{PORT}` | Public-facing URL used in server logs |

Example with custom values:

```bash
docker run -p 8080:8080 \
  -e PORT=8080 \
  -e HOST=0.0.0.0 \
  -e API_URL=https://your-domain.com \
  -t api
```

<Note>
  The built React app is embedded inside the Docker image at `/app/api/assets/`. No separate static file server or CDN is required — the Express server handles both the API routes under `/api` and the static assets under `/assets` from the same container.
</Note>
