---
title: "API server configuration and environment variables"
description: "Configure the PubKey UI Starter API server using HOST, PORT, and API_URL environment variables. Defaults work out of the box for local development."
---

The API server reads all configuration from environment variables at startup. The defaults are chosen so that no setup is required for local development — running `nx run api:serve` works without any `.env` file. For production deployments you will want to set at least `API_URL` to your public domain.

## ServerConfig interface

The `getServerConfig()` function returns a typed `ServerConfig` object that is passed directly to `server()`:

```typescript
interface ServerConfig {
  apiUrl: string
  host: string
  port: string
}
```

## Environment variables

<ParamField path="HOST" type="string" default="0.0.0.0">
  The network interface the server binds to. Use `0.0.0.0` to accept connections on all interfaces (required inside Docker or any container runtime). Set to `127.0.0.1` to restrict to localhost only.
</ParamField>

<ParamField path="PORT" type="string" default="3000">
  The TCP port the server listens on. The Dockerfile exposes this same default, so `docker run -p 3000:3000` works without changes.
</ParamField>

<ParamField path="API_URL" type="string" default="http://{HOST}:{PORT}">
  The public base URL of the server. This value is only used for logging at startup — it does not affect routing. Set it to your public domain in production so startup logs show the correct address.
</ParamField>

## getServerConfig function

The full implementation from `server-config.ts`:

```typescript server-config.ts
export interface ServerConfig {
  apiUrl: string
  host: string
  port: string
}

export function getServerConfig(): ServerConfig {
  const requiredEnvVars = [
    // Place any required environment variables here
  ]
  const missingEnvVars = requiredEnvVars.filter((envVar) => !process.env[envVar]?.length)

  if (missingEnvVars.length > 0) {
    console.error(`Missing environment variables: ${missingEnvVars.join(', ')}`)
    process.exit(1)
  }

  const host = process.env.HOST || '0.0.0.0'
  const port = process.env.PORT || '3000'

  const apiUrl = process.env.API_URL || `http://${host}:${port}`

  return {
    apiUrl,
    host,
    port,
  }
}
```

## Adding required environment variables

The `requiredEnvVars` array is where you declare variables that must be present for the server to start. If any are missing, the server logs the names and exits with code `1` before binding to any port.

```typescript server-config.ts
const requiredEnvVars = [
  'DATABASE_URL',
  'SECRET_KEY',
]
```

Add the variable name to the array, then read it from `process.env` below as usual:

```typescript
const databaseUrl = process.env.DATABASE_URL as string
```

## .env file example

```bash .env
HOST=0.0.0.0
PORT=3000
API_URL=http://localhost:3000
```

<Note>
  `dotenv` is loaded in `main.ts` before `getServerConfig()` is called, so `.env` files in the project root are automatically applied. You do not need any additional setup to use them during local development.
</Note>

<Tip>
  In production, set `API_URL` to your public domain (e.g., `https://api.example.com`). The value is printed to stdout at startup, which is useful for verifying deployments in CI logs or container orchestration dashboards.
</Tip>
